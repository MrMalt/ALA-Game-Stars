Cloud

De cloud kan het beste beschreven worden als een globaal netwerk van servers, elk met een unieke functie. De cloud is geen fysiek ding, maar een reusachtig netwerk van op elkaar aangesloten externe servers opveral op de wereld die samen in één ecosysteem fungeren. De servers zijn ontworpen voor het opslaan en beheren van gegevens, het uitvoeren van toepassingen of het leveren van content of services, zoals streaming video's, webmail, kantoorsoftware of sociale media. In plaats van dat u bestanden en gegevens van een lokale computer of pc haalt, opent u deze online vanaf elk apparaat met een internetverbinding. De informatie is overal en altijd toegankelijk.

A

  • AWS (Amazon)

    AWS is ’s werelds grootste cloud-serviceplatform. AWS combineert openheid en toegankelijkheid met een groot aanbod aan verschillende services en API’s.

    Een evoluerend, secured en uitgebreid cloud computing platform dat door Amazon wordt aangeboden en een mix van infrastructure as a service (IaaS), platform as a service (PaaS) en packaged software as a service (SaaS) combineert.

  • AWS CloudFormation

    AWS CloudFormation is een robuuste infrastructure as code tool die kan helpen bij het automatiseren en beheren van implementaties van AWS. CloudFormation templates kunnen naast JSON, ook met YAML gecreëerd worden.

    Gebruikers kunnen bovendien AWS CloudFormation Designer gebruiken om templates visueel te presenteren en de onderlinge dependencies van de resources te bekijken.

  • Azure (Microsoft)

    Azure is het cloudplatform van Microsoft dat verschillende internetdiensten aanbiedt voor bedrijven en ondernemingen. Je hoeft geen software te installeren, alle handelingen vinden online plaats. Met Azure kun je snel en gemakkelijk infrastructuren en services implementeren en daarmee is het één van de populairste cloud computing-platformen ter wereld.

B

  • BaaS (Backend as a Service)

    Backend as a Service (BaaS) is een cloud model dat dient als middelware en biedt ontwikkelaars handvatten om hun web- en mobiele applicaties te verbinden met de cloud. Het verbinden voltrekt zich door middel van API’s (Application Programming Interfaces) en SDK’s (Software Development Kits). In tegenstelling tot populaire cloud modellen zoals IaaS, PaaS en SaaS, is BaaS vrij nieuw en zijn er een beperkt aantal providers.

C

  • CaaS (Container as a Service)

    Containers as a service (CaaS) is zeer praktisch voor IT-afdelingen en ontwikkelaars bij het bouwen van veilige en schaalbare applicaties. Het verschaft gebruikers de mogelijkheid om containers, toepassingen en clusters te beheren en te implementeren op container gebaseerde virtualisatie. Met CaaS kan dit worden afgehandeld via de cloud of on-premise.

    In het kort, CaaS biedt een eenvoudige manier om een containercluster in te stellen en wordt vaak beschouwd als subset van IaaS. Echter, de container is de belangrijkste resource, in tegenstelling tot de virtual machine bij een IaaS cloud model.

  • Cloud ASP

    ASP staat voor Application Service Providing en betekent zoveel als het gebruiken of leveren van applicaties als dienst. De fysieke locatie van de software is bij de leverancier en de gebruikende organisatie logt via een beveiligde (internet) verbinding in. Met de beschikbaarheid van steeds hogere bandbreedtes tegen steeds lagere kosten, is Application Service Providing een serieus alternatief geworden.

  • Cloud Backup

    Cloud Backup (ook wel Backup as a Service of online backup genoemd) is een backup of kopie van locale files (op een werkstation, laptop of server) naar een data-schijf op het internet. Cloud Backup is goedkoop maar meestal niet redundant uitgevoerd: geen garanties over uptime en dataverlies.

  • Cloud Bridge

    Er is sprake van een 'cloud bridge' als een toepassing gebruikmaakt van componenten in verschillende cloud-omgevingen. Denk aan een toepassing die draait op een externe cloud en een deel van zijn gegevens uit een interne cloud haalt.

  • Cloud Burst

    Als een cloud niet beschikbaar is en je niet bij je data kunt (bijvoorbeeld door een piek in het gebruik), noemt men dat ook wel een 'Cloud Burst'.

  • Cloud computing

    Is een gestandaardiseerde IT-oplossing (diensten, software of infrastructuur) geleverd via het internet, waarbij uitsluitend betaald wordt voor wat wordt afgenomen (pay-per-use) en waarbij de klant zelf bepaalt wat hij afneemt (self-service).

  • Cloud OS

    Een besturingssysteem dat draait in de cloud, zoals Googles Chrome OS. Vergelijkbaar met Platform as a Service (PaaS).

  • Cloud Portability

    De mogelijkheid om toepassingen en data van de ene naar de andere cloud-omgeving te verplaatsen, met name als het gaat om clouds van verschillende leveranciers, maar ook bijvoorbeeld van interne naar externe clouds en andersom.

  • Cloud Provider

    Een dienstverlener die opslagruimte en/of rekenkracht (servers) in cloudvorm aanbiedt via een netwerk als het internet.

  • Cloud Service Architecture (CSA)

    Een ontwerp of architectuur waarin toepassingen of componenten van toepassingen als services vanuit de cloud beschikbaar worden gemaakt voor andere toepassingen.

  • Cloud Spanning

    Er is sprake van Cloud Spanning als een toepassing op meerdere cloud-omgevingen tegelijk draait.

  • Cloud Storage

    Opslag in de cloud. Cloud Storage (Storage as a Service of Online Disk) gaat verder dan Cloud Backup. In theorie onbeperkt schaalbaar, volledig redundant en optimaal beveiligd. Idealiter betaal je zelfs uitsluitend voor wat je afneemt. Bij Cloud Storage heeft een organisatie de beschikking over een extra 'schijf' die via internet (beveiligd) kan worden benaderd. Op deze schijf worden unieke bestanden geplaatst die niet ook nog ergens binnen het eigen netwerk staan. Daarom moet deze data extra beveiligd worden. Bij een goed uitgevoerde Cloud Storage hoort encryptie, redundancy en een off-site backup. Dat wil zeggen dat uitval van het systeem wordt voorkomen en dat de organisatie zelf kan kiezen welke backupconfiguratie men wil. Bovendien hoort de backup van de Cloud Storage in een ander datacenter te zijn ondergebracht voor maximale veiligheid.

  • Cloud Client

    Werkstation dat is geoptimaliseerd voor cloud computing, vergelijkbaar met een thin client. Thin client is een begrip uit de computerwereld. Het is een naam voor kleine, relatief zwakke computers die de hoofdmoot van het benodigde computercapaciteit van een server, een centrale computer betrekken en 1 gebruiker tegelijk "bedienen". De kleine computer fungeert daarbij als client ('klant') van de centrale computer.

  • Cloudcenter

    Een groot bedrijf dat zijn infrastructuur verhuurt voor cloudtoepassingen, zoals KPN. Via de site van Data Center Map vindt u data centra in Nederland als u een lokale service provider zoekt die wholesale space, colocatie rack space, dedicated servers, cloud servers, virtual servers, CDN, fiber, IP transit, ethernet services of andere hosting diensten aanbiedt.

  • Cloudsourcing

    Het uitbesteden van IT-diensten aan externe cloud-leveranciers, waarbij deze diensten zich gedragen alsof ze nog steeds intern gehost zijn.

  • Cloudstorm

    Er is sprake van een 'cloudstorm' als meerdere cloud-omgevingen met elkaar verbonden worden. Zie ook Cloud Spanning.

  • Cloudware

    Software waarmee het mogelijk is toepassingen te bouwen, uit te rollen, te draaien of te beheren in een cloudomgeving. Ook wel simpelweg alle software die in de cloud draait of kan draaien.

E

  • External Cloud

    Een cloud computing omgeving die buiten de grenzen van de eigen organisatie ligt. Niet per se hetzelfde als een publieke cloud: sommige externe Cloud Providers leveren uitsluitend specifieke diensten aan specifieke bedrijven.

F

  • FaaS (Function as a Service)

    Function as a service (FaaS) verwijst naar een cloud service die de ontwikkeling en het beheer van serverloze (serverless computing) applicaties mogelijk maakt. Serverless betekent niet dat er geen servers meer aan te pas komen. Wel betekent het dat gebruikers van FaaS kunnen programmeren (en andere taken) kunnen volbrengen, zonder zorgen over het inrichten en onderhouden van een server.

    Wanneer de klant een request doet, wordt een string van code getriggerd, in een container geladen en uitgevoerd door een externe server. Deze servers zijn in staat om de daarvoor beschreven functies uit te voeren, vandaar de naam; Functions as a Service.

    Zoals alle as a service toepassingen, is ook FaaS erop gericht om een hogere efficiëntie te behalen, tegen minder kosten. Kortom, FaaS is een toepassing van serverless computing: applicaties ontwikkelen zonder zorgen om een server te implementeren, aan te passen of te schalen. Het gehele proces wordt uitbesteed en is geautomatiseerd.

G

  • GCP (Google Cloud Platform)

    Google werkt aan een van de meest geavanceerde infrastructuren ter wereld om complexe zaken voor bedrijven snel te kunnen oplossen. Met Google Cloud Platform voorziet Google bedrijven van een groot aanbod aan services.

H

  • Hybrid Cloud

    Een hybride cloud is een combinatie van één of meerdere private en public cloud omgevingen, die met elkaar verbonden zijn. Het gebruik van deze cloud toepassing wordt momenteel erg veel toegepast. Een gedeelte van de bedrijfsapplicaties is dus actief in de private cloud en een ander deel van de infrastructuur draait in de public cloud.

    Onderling wisselen de verschillende cloud modellen data en gegevens uit, waardoor de gebruiker het overzicht houdt. Een Hybride cloud oplossing is goed geïntegreerd als je als gebruiker niet merkt in welke omgeving je werkzaam bent (in de public of de private cloud).

I

  • IaaS (Infrastructure as a Service)

    IaaS is een afkorting voor Infrastructure as a Service, ook wel Cloud infrastructure service genoemd. Bij deze manier van hosting wordt de infrastructuur, samen met alle netwerkelementen naar de cloud gebracht. Je houdt als gebruiker wel zelf het beheer op het operating systeem, de database, applicaties en data.

    IaaS maakt gebruik van Virtuele Machines (VM) en opslag vanaf een schaalbaar platform. Een groot voordeel van IaaS is dat de hardware (servers en netwerkapparatuur) in beheer blijft van de serviceprovider.

    Je gebruikt IaaS meestal op basis van een abonnement, waarbij je alleen betaalt wanneer je daadwerkelijk gebruikt maakt van de service. Hiermee lijkt IaaS nog het meest op de traditionele On-premise service, waarbij je de automatisering op eigen locatie beheert.

  • IBM Cloud

    Het IBM-cloudplatform combineert Platform as a Service (PaaS) met Infrastructuur as a Service (IaaS) om een ultieme geïntegreerde ervaring te bieden. Het platform schaalt en ondersteunt zowel kleine development teams en organisaties als grote enterprises.

    Het platform is gebouwd om ondersteuning te bieden aan allerlei gebruikersbehoeften, of het nu alleen private, public, hybride draait of gebruik maakt van een multicloud deployment. Met open-source technologieën, zoals Kubernetes, Red Hat OpenShift, en een compleet scala aan computing mogelijkheden, zoals virtuele machines, containers en serverless computing is er voor ieder wat wils.

O

  • On-premise cloud

    Met een on-premise cloud oplossing installeer je software lokaal, op de eigen server. Software lokaal installeren eist een flinke kapitaalinvestering. Eenmaal geïnstalleerd zijn er systeembeheerders nodig voor beheer en onderhoud. Wanneer veranderingen benodigd zijn, dan ben je zelf verantwoordelijk voor de uitvoering.

P

  • PaaS? (Platform as a Service)

    PaaS is een afkorting voor Platform as a Service, ook Cloud Platform Service genoemd. Met het gebruik van deze hosting richt je snel en gemakkelijk een platform in naar je eigen behoefte en wensen. Zo kun je PaaS gebruiken voor de inrichting van een databaseserver of een webserver.

    PaaS is een computing platform die is ontworpen om organisaties, in het bijzonder ontwikkelaars, te helpen de voordelen van cloud computing te realiseren. Je kunt het platform gebruiken om eenvoudig software te ontwikkelen, te testen en in productie te brengen. Het grote voordeel hiervan is dat je geen omkijken hebt naar het onderhoud van de software en infrastructuur van het netwerk.

  • Private cloud

    Een private cloud (ook wel interne cloud of corporate cloud genoemd) is een omgeving buiten je eigen netwerkserver, die alleen beschikbaar is voor je eigen organisatie. Het belangrijkste kenmerk van de private cloud is dat er geen hard- en software wordt gedeeld met andere organisaties.

  • Public cloud

    Bij een public cloud omgeving wordt gebruik gemaakt van een gedeelde infrastructuur die voor iedereen toegankelijk is. Deze cloud-omgeving vereist geen investeringen in hard- en software. Voor een vast bedrag per maand kun je een applicatie en ruimte van één of meerdere cloud-leveranciers huren en daar je bestanden of data opslaan. Verder kan je als klant geen invloed uitoefenen op de infrastructuur, de functionaliteit of de kwaliteit van de cloud.

    Het maakt niet uit waar de hardware staat waarop de data wordt opgehaald en opgeslagen; dit kan overal ter wereld zijn. De public cloud-leverancier zorgt voor het onderhoud en de updates aan de infrastructuur. Over het algemeen betaal je voor de hoeveelheid capaciteit die je gebruikt. Bekende public cloud-providers zijn: Amazon Web Services (AWS), Azure, GCP en ICP.

S

  • SaaS (Software as a Service)

    SaaS (Software as a Service) biedt software (applicaties) online als dienst of service aan. SaaS wordt ook wel cloud software, ASP of software on demand genoemd. De eerste bekende applicaties voor consumenten waren webmail, bladwijzers en het delen van foto’s.

    Een SaaS oplossing wordt door bedrijven en instellingen het vaakst gebruikt als cloud oplossing. De gehele applicatie wordt geïnstalleerd, onderhouden en beheerd door een Saas-leverancier. Daarnaast liggen updates en bugfixes aan de applicatie in de handen van de leverancier.

    Een cloud-omgeving die is geoptimaliseerd voor gebruik door een specifieke (verticale) branche, industrie of toepassing.

  • Virtual Private Cloud (VPC)

    Als een VPN (Virtual Private Network) maar dan voor cloud computing. Een afgescheiden, beveiligd gedeelte binnen een (publieke) cloud-omgeving.
